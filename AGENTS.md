# Improving and reviewing the skills

When ask by user, do a review of the selected skills according to the
per skill topic guidelines.

## ADR related skills

Skills related to current topic:

- skills/architecture-decision-record/SKILL.md

- skills/architecture-decision-record-review/SKILL.md

Re-read and review the skills for
- whether an agent can understand them
- consistency among all skills
- report issues
- propose improvments
- check whether the guidelines of one skill to use another skill are
  good and understandable by an agent
- check for consistent output formatting guidelines within the skills
- check whether authoratative status of `architecture-decision-record` skill
  is communicated well, so no structural content of an ADR needs to be
  duplicated in the other skills

While checking the skills, always remember that
`architecture-decision-record` skill is the canonical ADR validator!

Create a `todo.md` with list of issues/proposals per skill
that can be checkmarked.

Maintain the todos while you or user fixes issues/proposals.
